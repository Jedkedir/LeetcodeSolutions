# LeetCode Solutions

A curated collection of LeetCode problem solutions organized by problem number and title.

## Repository structure

Each problem lives in its own folder named with the LeetCode problem number and a kebab-case title. Inside each folder you will typically find:

- A short README describing the problem and approach
- One or more solution files (currently Python)

Example:

- 1-two-sum/
  - README.md
  - two-sum.py

## How to use

- Browse by problem number in the top-level directory
- Open the problem folder to view the solution and notes

## Conventions

- Folder names follow the format: {number}-{problem-title}
- Solution files are named after the problem title

## Contributing

Contributions are welcome. If you add a new solution:

- Follow the existing folder and file naming conventions
- Include a brief README for the problem
- Keep the solution clear and well-commented
